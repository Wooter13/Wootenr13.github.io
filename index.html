Types of Topologies

Overview

There are moany different types of topolgies that people use to create their network. Every different type of network has their advantages and disadvantages. here is a brief discription of some of the more common types of networks.

Star Topology

The star topology is used for local area networks, where all end devices are connected to a central device individually. Since each connection is individual this type of topology does use more cables. However, if one link fails it does not effect and of the other connections. Still every setup has a disadvantage. With this topology there is a lot of dependency on the central node meaning it is using a lot of resources. If the central node were to go down then the whole network would not be able to communicate with each other.

Full Mesh Topology

In a full mesh topology every node in the network is connected to each other. This type of topology can be implemented on either a local area network, a wireless loca area network, or a virutal local are network. Having this set-up means if one connection goes down it will be re-routed, so the node still is connected to the network. However, this type of topology can become very hard to manage if too many nodes are added due to all of the connections. It will require constant supervision.

Bus Topology

With a bus topology every node in the network is connected to a main cable or link called a bus. This is a simple yet still reliable setup, when one link goes down the rest are still able to communicate with each other. Another benefit to this topolgy is it is scalable. Additional nodes can be added anywhere along the bus. A disadvantage to this toplogy is if the main bus goes down then the whole network is down, and can not communicate with anyone until the bus is operational again.
